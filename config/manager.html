# 服务接口

# REST v1

# REST v2

新研的服务接口，以/igs/rest/services开头，MapServer、TileServer这类接口

# 服务实例

服务实例为服务管理中服务管理最小单元，一个服务实例对应一个GIS服务

# 服务实例v1

对应REST v1的服务接口，服务接口类型没有清晰的区分，主要是通过服务对应的发布数据源来确定，比如地图文档服务

# 服务实例v2

对应REST v2的服务接口

注意，目前使用服务管理v2的版本，也兼容了REST v1的接口

# 服务标识

服务标识（一个GIS服务器中确保唯一性）在服务实例v1中使用服务名，在服务实例v2中使用文件夹名+服务名

  • 文件夹名：只能包括中文字母数字-_，且不能以-或数字开头

  • 服务名：只能包括中文字母数字-_，且不能以-或数字开头

注意：服务名受制于OGC中XML标签名规范，因此不能包含特殊字符

在服务实例v2中，兼容REST v1服务时，服务名=[文件夹:]服务名，OGC中xml标签使用[文件夹_]服务名（注意：xml标签不能使用':'，因此改用'_')
